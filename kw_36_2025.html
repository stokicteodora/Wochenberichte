JSON: Grundlagen, Aufbau und Anwendung

JSON

JSON (JavaScript Object Notation) ist ein textbasiertes Datenformat, das Informationen in einer klar strukturierten und leicht lesbaren Form darstellt. Es wurde Ende der 1990er-Jahre von Douglas Crockford spezifiziert und hat sich seither als Standard für den Datenaustausch etabliert.

Wie funktioniert JSON?

Ein JSON-Dokument beginnt und endet immer mit geschweiften Klammern { }. Innerhalb dieser Klammern werden Objekte definiert, die aus Name/Wert-Paaren bestehen. Ein Name (oder Schlüssel) wird dabei durch einen Doppelpunkt : vom Wert getrennt. Der Wert kann wiederum ein einfaches Element, ein weiteres Objekt oder ein Array sein.

  • Boolean: Wahrheitswerte true oder false.
  • Zahlen: Ganzzahlen, Dezimalzahlen oder Exponentialzahlen.
  • Strings: Text in doppelten Anführungszeichen.
  • Null: Repräsentiert die Abwesenheit eines Werts.

JSON basiert auf zwei grundlegenden Strukturen: Objekte und Arrays. Objekte sind ungeordnete Sammlungen von Name/Wert-Paaren, die in geschweifte Klammern eingeschlossen sind und durch Kommas voneinander getrennt werden. Arrays hingegen sind geordnete Listen von Werten, die in eckige Klammern gesetzt und ebenfalls durch Kommas getrennt werden. Werte in JSON können dabei beliebig verschachtelt sein, sodass ein Array Objekte enthalten kann oder ein Objekt selbst wieder Arrays beinhaltet.

Beispiel


          {
          "name": "Fiona Fischer",
          "alter": 33,
          "verheiratet": true,
          "kinder": ["Anna", "Nik"],
          "adresse": {
          "strasse": "Beispielstrasse1",
          "stadt": "Beispielstadt",
          "plz": "12345" 
          }
        }
  • name ist ein String
  • alter ist eine Zahl
  • verheiratet ist ein Boolean
  • kinder ist ein Array
  • adresse ist ein verschachteltes Objekt

Verwendung

JSON wird vor allem genutzt, um Daten übersichtlich zu speichern und zwischen Programmen auszutauschen. In Webanwendungen hilft es, Informationen vom Browser zum Server zu senden und umgekehrt, und erleichtert so die Zusammenarbeit verschiedener Programmiersprachen.

Vor- und Nachteile von JSON

JSON ist ein beliebtes Format für den Datenaustausch, weil es leicht zu lernen und einfach zu benutzen ist. Die Syntax ist übersichtlich, sodass die Daten sowohl für Menschen als auch für Maschinen gut lesbar sind.

Vorteile von JSON:

  • Leicht verständlich und einfach zu schreiben: Die Struktur ist übersichtlich, wodurch Einsteiger sie schnell lernen können.
  • Programmiersprachenunabhängig: JSON kann mit fast allen Programmiersprachen genutzt werden.
  • Schnell und effizient:JSON-Daten lassen sich leicht übertragen und verarbeiten.
  • Flexibel einsetzbar: Es eignet sich für viele Anwendungen und Systeme.

Nachteile von JSON:

  • Grössere Dateigrösse: JSON benötigt manchmal mehr Speicherplatz als Binärformate.
  • Bestimmte Datentypen fehlen: Z.B. Datum oder Binärdaten können nicht direkt gespeichert werden.
  • Keine automatische Datenvalidierung: JSON prüft die Daten selbst nicht auf Fehler.

Fazit

JSON ermöglicht eine klare Strukturierung und den einfachen Austausch von Daten zwischen Programmen. Es sorgt für effiziente Speicherung und eine zuverlässige Kommunikation in Webanwendungen
